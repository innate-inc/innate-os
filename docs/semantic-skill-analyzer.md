# Semantic Skill Analyzer

The `semantic_skill_analyzer` package cross-references a natural-language scenario against the robot's existing capabilities and produces a structured JSON file describing what is already covered and what needs to be built.

It uses a local [ollama](https://ollama.com) instance with `qwen3:0.6b` and has no dependency on brain_client, ROS, or any other innate-os component.

## Prerequisites

| Requirement | Notes |
|-------------|-------|
| ollama running locally | `ollama serve` (default port 11434) |
| `qwen3:0.6b` model pulled | `ollama pull qwen3:0.6b` |
| `ollama` Python package | included in `ros2_ws/pip-requirements.txt`; already present in the `local_llm` conda env |
| `~/.wildrobot/capabilities.json` | generated by `scripts/export_capabilities.py` or automatically on robot restart |

## How it works

1. `analyze()` loads `~/.wildrobot/capabilities.json` — the index of all agents and their skills.
2. It sends the user scenario and the capabilities JSON to the LLM and asks for a JSON response directly (no tool calling — small models ignore tool schemas and produce empty output).
3. Thinking mode (`think=True`) is enabled so the model can reason before writing the JSON.
4. Deterministic post-processing fixes common hallucinations using the capabilities index as ground truth.
5. The result is written to `~/.wildrobot/<uuid>-missing-skills.json` and the path is returned.

## Output format

```json
{
  "existing_agents": ["demo_agent"],
  "missing_capabilities": {
    "visitor_monitor_agent": {
      "prompt": "Detailed description of what this agent does and which skills it calls.",
      "new_skills": [
        {"detect_face": "Detect and identify a human face using the robot camera."},
        {"log_arrival": "Write a timestamped visitor entry to a local log file."}
      ]
    }
  }
}
```

- **`existing_agents`** — the single best-matching existing agent that covers the request (selected by keyword overlap against agent ID and skill names). Empty when nothing relevant exists.
- **`missing_capabilities`** — at most one new agent that needs to be created, with a detailed prompt and a list of new skills. Empty when everything is already covered.

## Post-processing guarantees

The LLM output is filtered deterministically after every call — no prompt engineering required:

| Hallucination | Fix |
|---------------|-----|
| Existing agent listed as missing | Removed (exact match against capabilities index) |
| Skill basename listed as new agent name | Removed (basenames extracted from capabilities index) |
| Multiple missing agents returned | Trimmed to the first entry |
| Existing agents with no keyword overlap to the prompt | Discarded (token overlap score must be > 0) |
| Multiple existing agents returned | Narrowed to the single best overlap match |

## Retry on blank response

If the model returns both fields empty (tool call ignored, content unparseable), `analyze()` automatically retries once with a nudge message appended to the conversation. `max_retries` defaults to `1` and can be overridden:

```python
output_path = analyze(prompt, max_retries=2)
```

## Python API

```python
from semantic_skill_analyzer import analyze, load_capabilities

# Run gap analysis — returns path to the written file
output_path = analyze("I want the robot to greet visitors and log their arrival time")

# Inspect the result
import json, pathlib
result = json.loads(pathlib.Path(output_path).read_text())
print(result["existing_agents"])       # best-matching existing agent (list of 0 or 1)
print(result["missing_capabilities"])  # new agent/skills to build (0 or 1 entry)

# Load the capabilities index directly (no LLM call)
caps = load_capabilities()  # reads ~/.wildrobot/capabilities.json
```

### `analyze()` parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `prompt` | `str` | required | Natural-language scenario to analyze |
| `capabilities_path` | `str` or `Path` | `~/.wildrobot/capabilities.json` | Capabilities index to compare against |
| `output_dir` | `str` or `Path` | `~/.wildrobot` | Directory for the output file |
| `ollama_host` | `str` | `http://localhost:11434` | Ollama API endpoint |
| `max_retries` | `int` | `1` | Retries when the model returns a blank result |

Returns the absolute path of the written `<uuid>-missing-skills.json` file.

## Test scripts

### Script 1 — gap analysis (produces a missing-skills file)

Use this to explore what new agent needs to be built for a scenario the robot cannot yet handle.

```bash
conda run -n local_llm python3 scripts/test_skill_gap_analysis.py
# or with a custom prompt:
conda run -n local_llm python3 scripts/test_skill_gap_analysis.py "Call my mum on WhatsApp and order a pizza"
```

Expected output:
```
Covered by existing agents:
  (none)

Missing capabilities:

  [New agent] whatsapp_pizza_agent
  Prompt: Agent that places a WhatsApp call and handles a food order on behalf of the user.
  New skills:
    • make_whatsapp_call: Initiate a WhatsApp voice call to a contact by name or number.
    • order_food_online: Place a food delivery order and return the estimated delivery time.
```

### Script 2 — coverage check (verifies existing capabilities)

Use this to confirm that a fully-covered request is identified correctly and no new agents are proposed.

```bash
conda run -n local_llm python3 scripts/test_capabilities_covered.py
# or with a custom prompt:
conda run -n local_llm python3 scripts/test_capabilities_covered.py "Navigate to position 1.5, 2.0 and wave"
```

Expected output:
```
✓ Request is fully covered by existing agents:
    demo_agent
```

## Unit tests

Tests run without a live ollama instance or any ROS dependency. `ollama.Client` is replaced with a mock.

```bash
conda run -n local_llm python3 -m pytest semantic_skill_analyzer/tests/ -v
```

## Integration with innate-os components

Since `INNATE_OS_ROOT` is added to `sys.path` at runtime by `agent_loader.py`, any innate-os component can import the library directly:

```python
from semantic_skill_analyzer import analyze

output_path = analyze(user_prompt)
```

No additional path setup is required when running inside the ROS environment.

## Updating the capabilities index

The capabilities index is regenerated automatically every time the robot restarts (via `initialize_agents()` in `brain_client/initializers.py`). To regenerate it manually without restarting:

```bash
# With ROS workspace sourced (full skill descriptions):
python3 scripts/export_capabilities.py

# Without ROS (skill basenames only, no descriptions):
conda run -n local_llm python3 scripts/export_capabilities.py
```

Previous snapshots are preserved as `capabilities.json.<ISO8601>` in `~/.wildrobot/`.

## Model quality

`qwen3:0.6b` is a 600M-parameter model — fast and lightweight but limited in reasoning depth. Output quality improves significantly with a larger variant:

```bash
ollama pull qwen3:4b
```

Then change `MODEL` in `semantic_skill_analyzer/analyzer.py`:

```python
MODEL = "qwen3:4b"
```

No other code changes are needed. All post-processing and retry logic is model-agnostic.
